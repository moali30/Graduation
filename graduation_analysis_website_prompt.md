# Prompt: بناء موقع التحليل الإحصائي لمشاريع التخرج

---

## نظرة عامة على المشروع

أنت مساعد متخصص في بناء تطبيقات الويب. مهمتك هي بناء **موقع ويب متكامل للتحليل الإحصائي لاستبيانات مشاريع التخرج**، موجّه للطلاب والباحثين. الموقع يتيح للمستخدم رفع ملف بيانات، تعريف المتغيرات، ثم تشغيل تحليل إحصائي احترافي وشامل بنقرة واحدة.

**Backend:** Appwrite (Auth + Storage + Databases + Functions)  
**Frontend:** React + TypeScript + TailwindCSS  
**مكتبات الرسوم البيانية:** Recharts أو Chart.js  
**اللغة الافتراضية للواجهة:** العربية مع دعم كامل للمصطلحات الإنجليزية الإحصائية

---

## متطلبات Appwrite

### Collections المطلوبة (Database)
```
projects {
  id, user_id, title, description, file_id, status, created_at
}

variable_definitions {
  id, project_id, variable_name, columns[], type (dependent|independent|demographic), scoring_method (mean|sum)
}

analysis_results {
  id, project_id, cronbach_alpha[], descriptive_stats{}, correlation_matrix{}, regression_results{}, created_at
}
```

### Storage Buckets
- `survey_files` — لرفع ملفات الإكسيل الأصلية
- `analysis_reports` — لحفظ تقارير PDF وملفات الإكسيل الناتجة

### Appwrite Functions
- `parse-excel` — تحليل ملف الإكسيل واستخراج أسماء الأعمدة والبيانات
- `run-analysis` — تشغيل خط التحليل الكامل (Python runtime)
- `generate-optimized-data` — توليد البيانات المُصلَحة
- `export-report` — تصدير التقرير النهائي بصيغة Excel

## تفاصيل كل خطوة

---

### الخطوة 1: رفع الملف

**واجهة المستخدم:**
- منطقة Drag & Drop لرفع الملف (يقبل xlsx, xls, csv فقط)
- يعرض اسم الملف وحجمه بعد الرفع
- زر "معاينة البيانات" يفتح Modal يعرض أول 10 صفوف من الملف في جدول
- يعرض قائمة بجميع أسماء الأعمدة المستخرجة من الملف

**المعالجة الخلفية (Appwrite Function: `parse-excel`):**
- رفع الملف إلى `survey_files` bucket في Appwrite Storage
- استدعاء الدالة `parse-excel` التي تُعيد:
  ```json
  {
    "columns": ["Q1", "Q2", "Gender", "Age", ...],
    "row_count": 150,
    "preview": [[...], [...], ...],
    "data_quality_report": {
      "missing_values_per_column": {"Q1": 3, "Q2": 0, ...},
      "non_numeric_in_columns": {"Q3": ["غير متاح", "N/A"]},
      "out_of_range_values": {"Q5": [6, 7, 0]}
    }
  }
  ```

**تقرير جودة البيانات (Data Quality Report):**
يُعرض تلقائياً بعد رفع الملف في بطاقة تحذيرية واضحة:

| نوع المشكلة | الأعمدة المتأثرة | الإجراء المقترح |
|---|---|---|
| قيم مفقودة (Missing Values) | Q1: 3 قيم، Q4: 7 قيم | سيتم الاستبدال بالمتوسط تلقائياً |
| قيم نصية في عمود رقمي | Q3: يحتوي "N/A", "غير متاح" | يجب استبدالها بأرقام (1-5) |
| قيم خارج النطاق (1-5) | Q5: يحتوي على 6, 7, 0 | يجب تصحيحها لتكون بين 1 و 5 |
| صفوف مكررة تماماً | 4 صفوف مكررة | سيتم حذفها تلقائياً |

**قواعد التحقق من صحة البيانات:**
- عمود ليكرت صالح: كل قيمه أرقام بين 1 و 5 فقط
- إذا وُجدت قيم نصية → تحذير أحمر مع التوجيه الدقيق
- إذا وُجدت قيم خارج نطاق 1-5 → تحذير برتقالي
- إذا كانت القيم المفقودة > 20% في عمود → تحذير خاص
- إذا كان عدد الصفوف < 30 → تحذير بأن العينة صغيرة جداً وقد تضعف النتائج

---

### الخطوة 2: تعريف المتغيرات

**القسم أ: المتغيرات الديموغرافية**

- يعرض جميع أعمدة الملف كـ Chips قابلة للنقر
- المستخدم يضغط على الأعمدة الديموغرافية لتحديدها (Gender, Age, Education, ...)
- لكل عمود محدد، يظهر Dropdown لتحديد نوعه:
  - **فئوي (Categorical)** مثل: الجنس، المؤهل، التخصص
  - **عددي مستمر (Continuous)** مثل: العمر، سنوات الخبرة

**القسم ب: متغيرات الدراسة (أسئلة ليكرت)**

واجهة بطاقات ديناميكية. لكل متغير، يتم إنشاء بطاقة تحتوي على:

```
┌─────────────────────────────────────────────┐
│  اسم المتغير: [رضا_العملاء]                  │
│                                             │
│  الأسئلة المرتبطة: [Multi-select Dropdown]  │
│  ☑ Q1  ☑ Q2  ☑ Q3  ☐ Q4                   │
│                                             │
│  نوع المتغير:                               │
│  ● مستقل (X)    ○ تابع (Y)                 │
│                                             │
│  طريقة التجميع:                             │
│  ● المتوسط (Mean)   ○ المجموع (Sum)         │
│                                          ✕  │
└─────────────────────────────────────────────┘
```

- زر "+ إضافة متغير جديد" يضيف بطاقة جديدة فارغة
- التحقق الفوري: يمنع تعيين نفس العمود لمتغيرين مختلفين
- يعرض ملخصاً في الشريط الجانبي: عدد المتغيرات المستقلة / التابعة / الديموغرافية

**التحقق من صحة تعريف المتغيرات:**
- يجب أن يكون هناك متغير تابع واحد على الأقل
- يجب أن يكون هناك متغير مستقل واحد على الأقل
- لا يجوز ترك اسم المتغير فارغاً
- كل متغير يجب أن يضم عموداً واحداً على الأقل

**إعدادات المعالجة العامة (يظهر في أسفل الخطوة):**
```
طريقة التعامل مع القيم المفقودة:
● الاستبدال بالمتوسط (Mean Imputation) — موصى به
○ حذف الصفوف (Listwise Deletion)

مستوى الدلالة الإحصائية (α):
● 0.05   ○ 0.01   ○ 0.10
```

## صفحة النتائج `/project/:id/results`

### القسم 1: ملخص التحليل (Dashboard)

4 بطاقات سريعة في الأعلى:
- عدد المبحوثين (N)
- متوسط ألفا كرونباخ للمتغيرات
- R² للنموذج
- مستوى دلالة النموذج (p-value)

---

### القسم 2: تحليل الثبات والموثوقية

**Cronbach's Alpha لكل متغير:**

| المتغير | عدد الأسئلة | ألفا كرونباخ | التفسير |
|---|---|---|---|
| رضا_العملاء | 4 | 0.87 | ✅ ممتاز (> 0.80) |
| جودة_الخدمة | 5 | 0.74 | ✅ مقبول (> 0.70) |
| الولاء | 3 | 0.61 | ⚠️ منخفض (< 0.70) |

**تفسير تلقائي** يظهر أسفل الجدول:
- α ≥ 0.90: ممتاز — Excellent
- 0.80 ≤ α < 0.90: جيد جداً — Good
- 0.70 ≤ α < 0.80: مقبول — Acceptable
- α < 0.70: يحتاج إلى مراجعة الأسئلة — Poor

---

### القسم 3: التحليل الوصفي

#### 3أ: الإحصاءات الوصفية للمتغيرات الأساسية

جدول يعرض: الاسم، N، المتوسط (Mean)، الانحراف المعياري (SD)، الوسيط (Median)، الحد الأدنى، الحد الأقصى

**تفسير تلقائي للمتوسطات** بناءً على مقياس ليكرت 1-5:
- 1.00 – 1.80: منخفض جداً
- 1.81 – 2.60: منخفض
- 2.61 – 3.40: متوسط
- 3.41 – 4.20: مرتفع
- 4.21 – 5.00: مرتفع جداً

#### 3ب: التحليل الوصفي للمتغيرات الديموغرافية

لكل متغير ديموغرافي:
- **جدول التكرارات والنسب المئوية** (Frequency & Percentage)
- **رسم بياني عمودي (Bar Chart)** بالألوان — مع إمكانية تحميله كصورة PNG
- **فطيرة (Pie Chart)** للمتغيرات الفئوية

---

### القسم 4: تحليل الارتباط

- **مصفوفة الارتباط (Correlation Matrix)** كجدول مع تلوين الخلايا (Heatmap-style):
  - أخضر داكن: ارتباط موجب قوي (r > 0.7)
  - أخضر فاتح: ارتباط موجب معتدل
  - أحمر: ارتباط سلبي
  - رمادي: ارتباط ضعيف
- تعليق تلقائي على أقوى وأضعف الارتباطات
- قيمة Pearson r ومستوى الدلالة p لكل خلية

---

### القسم 5: تحليل الانحدار الخطي

لكل متغير تابع (Y):

#### 5أ: ملخص النموذج (Model Summary)

| المؤشر | القيمة | التفسير |
|---|---|---|
| R | 0.812 | قوة الارتباط المتعدد |
| R² | 0.660 | النموذج يفسر 66% من التباين |
| R² المعدّل | 0.648 | — |
| F | 54.23 | — |
| p-value | < 0.001 | ✅ النموذج دال إحصائياً |

#### 5ب: معاملات الانحدار

| المتغير المستقل | B (غير معياري) | Beta (معياري) | t | p-value | الدلالة |
|---|---|---|---|---|---|
| جودة_الخدمة | 0.412 | 0.389 | 4.21 | < 0.001 | ✅ دال |
| سرعة_التسليم | 0.298 | 0.271 | 3.17 | 0.002 | ✅ دال |
| التواصل | 0.118 | 0.095 | 1.23 | 0.220 | ✗ غير دال |

#### 5ج: تشخيصات الافتراضات (Diagnostics)

**التعددية الخطية (Multicollinearity - VIF):**

| المتغير | VIF | الحكم |
|---|---|---|
| جودة_الخدمة | 2.14 | ✅ مقبول (< 5) |
| سرعة_التسليم | 1.87 | ✅ مقبول |
| التواصل | 6.42 | ⚠️ تعددية خطية عالية (> 5) |

**الرسوم البيانية التشخيصية (تُعرض بجانب بعض):**
1. **Residuals vs Fitted** — لاختبار افتراض التجانس (Homoscedasticity)
2. **Normal Q-Q Plot** — لاختبار افتراض الاعتدالية (Normality)

كلا الرسمين قابلان للتنزيل.

## صفحة تصليح البيانات `/project/:id/fix`

> هذه الصفحة تظهر تلقائياً كاقتراح عند اكتشاف أي من هذه الحالات:

> - التعددية الخطية (VIF > 5)

### تفاصيل ما يحدث خلف الكواليس (يُعرض بشكل مبسط)

- يتم إنشاء بيانات ليكرت (1-5) تتبع التوزيع الطبيعي
- يتم ضمان ارتباطات منطقية وواقعية بين المتغيرات
- يتم التحكم في نسبة الضوضاء (Noise) لتجنب نموذج مثالي بالكامل (R² = 100%)
- يتم الحفاظ على التوزيع الأصلي للمتغيرات الديموغرافية
- يتم ضمان ألفا كرونباخ ≥ 0.80 لجميع المتغيرات

- يُعرض التحليل الكامل على البيانات الجديدة تلقائياً

## اعتبارات الأمان والخصوصية

- المصادقة إلزامية (Appwrite Auth) لحفظ المشاريع
- يمكن استخدام "الوضع الضيف (Guest)" بدون تسجيل لكن بدون حفظ النتائج
- كل مستخدم لا يرى إلا مشاريعه الخاصة (Row-level security في Appwrite)
- الملفات المرفوعة محمية بـ Appwrite Storage permissions
- يُمسح الملف من Storage بعد اكتمال التحليل وحفظ النتائج في Database (لتوفير المساحة)

---

## ملاحظات التنفيذ الجوهرية

1. **جميع الحسابات الإحصائية** يجب تنفيذها في Appwrite Functions (Python runtime) — ليس في المتصفح
2. الدالة `run-analysis` يجب أن تُعيد كائن JSON كامل يحتوي على جميع النتائج لتخزينها في `analysis_results` collection
3. استخدم WebSockets أو Appwrite Realtime لتحديث Progress Bar في الوقت الحقيقي أثناء التحليل
4. الرسوم البيانية تُولَّد في الـ Frontend من البيانات الرقمية المُعادة من الـ Backend (ليس صوراً جاهزة)
5. **معامل التجميع الافتراضي**: المتوسط (Mean) — مع تنبيه أن Sum مناسب فقط للدراسات التي تستخدم المجموع الكلي للمقياس

## خلاصة خط التحليل الكامل (Pipeline)

```
رفع الملف
    ↓
قراءة الأعمدة → عرض تقرير جودة البيانات
    ↓
تعريف المتغيرات (ديموغرافية / مستقلة / تابعة)
    ↓
تحويل أعمدة ليكرت إلى أرقام + معالجة القيم المفقودة
    ↓
حساب Cronbach's Alpha لكل متغير
    ↓
بناء المتغيرات المركبة (Composite Variables) بالمتوسط
    ↓
التحليل الوصفي:
  ├─ إحصاءات المتغيرات الأساسية (Mean, SD, Min, Max)
  └─ تكرارات ورسوم الديموغرافيات
    ↓
تحليل الارتباط (Pearson Correlation Matrix + Heatmap)
    ↓
الانحدار الخطي المتعدد (OLS Regression):
  ├─ R², Adj R², F, p-value
  ├─ معاملات B, Beta, t, p لكل متغير
  └─ تشخيصات: VIF + Q-Q Plot + Residuals vs Fitted
    ↓
[إذا كانت النتائج ضعيفة]
    ↓
تصليح البيانات (توليد بيانات محسّنة)
    ↓
إعادة التحليل الكامل على البيانات الجديدة
    ↓
تصدير التقرير (Excel + نص أكاديمي)
```
